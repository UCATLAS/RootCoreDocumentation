00025 Methods used in combiner tools. The problem is to generate unique 
00026 set of N indeces from M containers in a general way. For eg, suppose
00027 there are three containers each containing 3, 4, 5 elements respectively:
00028 one way to solve it is to construct an "odometer" each digit going up
00029 to the number of elements. Of cource, uniqueness of each combinations
00030 should be ensured. The problem is that some digits of the odometer
00031 may come from the same container. For the problem to be solved by odometer
00032 approach (is a useful approach to be able to specify a combination by 
00033 ["jet" "jet" "lepton"] etc), digits that come from the same container
00034 always have smaller number on the left.
00035 
00036 eg. if the combination is ["jet", "jet", "lepton", "bjet", "lepton"]
00037 (problem is simplified if repeated labels always come together but
00038 this cannot be assumed)
00039 and "jet" container has 4 elements, "lepton" 2, and "bjet" 2, then 
00040 00000 is not acceptable since two jets and leptons chosen are the same.
00041 01001 is the smallest acceptable number.
00042 01101 is not acceptable since leptons are the same
00043 01100 is not acceptable since the lepton digits 21 an 12 are indifferent
00044       in terms of combination and 01102 has already been counted.
00045 01102 won't be counted since there's only two leptons.
