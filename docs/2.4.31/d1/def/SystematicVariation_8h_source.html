14 // This module implements a class that manages the various
15 // representations of a single systematic variation. The interface
16 // provided in this module is intended for the general user. The
17 // module is considered to be in the pre-alpha stage.
18 
19 // The current implementation isn't particularly efficient, in fact it
20 // is terribly inefficient, as most of the accessors involve a lot of
21 // string operations. However, the current accessors should allow for
22 // more efficient implementations if needed. And this class is likely
23 // only to be used during job startup, meaning an inefficient
24 // implementation is likely to have little to no effect on overall job
25 // execution time. At the same time objects of this type are likely
26 // to be kept around for the entire runtime of the job, meaning the
27 // current small memory footprint should have advantages.
