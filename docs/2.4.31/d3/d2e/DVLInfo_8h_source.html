59 /*
60  * @brief Holder to implement conversion copies for @c DataVector/@c DataList.
61  *
62  * In some cases, we want to convert @c DataVector<A> to @c DataVector<B>,
63  * where the two @c DataVector classes are not related by inheritance,
64  * but where @c A and @c B are. In such a case, we can make a new view
65  * container for @c DataVector<B> and fill it with the pointers
66  * from @c DataVector<A> (suitably converted). In order for this
67  * to work with the conversion machinery in SGTools, we must be able
68  * to do this given only the @c type_info for @c DataVector<B>.
69  * Thus, we build a registry of @c DVLInfo objects, indexed
70  * by the @c type_info; the @c DVLInfo objects have virtual methods
71  * to build the new container and to fill it. (Everything said here
72  * for @c DataVector also applies to @c DataList.)
73  *
74  * Note: these objects should only be allocated statically.
