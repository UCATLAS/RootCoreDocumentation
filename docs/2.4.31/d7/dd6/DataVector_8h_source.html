563 // These are the intermediate classes from which @c DataVector derives
564 // in the case of multiple or virtual derivation. The actual
565 // definitions are in the @c icc file. They have the following
566 // properties:
567 // - They derive from those of @c B1, @c B2, and @c B3 that are not
568 // @c NoBase.
569 // - They perform (static) checking to ensure that there's a unique
570 // base class in the hierarchy.
571 // - They make the following types available from the base classes:
572 // - @c PtrVector
573 // - @c BaseContainer
574 // - @c size_type
575 // - @c difference_type
576 // - @c allocator_type
577 // - @c has_virtual is defined as @c true.
578 // - An override of @c dvlinfo_v is defined, to prevent ambiguities.
