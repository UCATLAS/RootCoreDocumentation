00014 // This module implements a class that manages the various
00015 // representations of a single systematic variation.  The interface
00016 // provided in this module is intended for the general user.  The
00017 // module is considered to be in the pre-alpha stage.
00018 
00019 // The current implementation isn't particularly efficient, in fact it
00020 // is terribly inefficient, as most of the accessors involve a lot of
00021 // string operations.  However, the current accessors should allow for
00022 // more efficient implementations if needed.  And this class is likely
00023 // only to be used during job startup, meaning an inefficient
00024 // implementation is likely to have little to no effect on overall job
00025 // execution time.  At the same time objects of this type are likely
00026 // to be kept around for the entire runtime of the job, meaning the
00027 // current small memory footprint should have advantages.
